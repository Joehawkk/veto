# Veto — MVP Specification

## Цель MVP

Веб-приложение, где пользователь вводит товар и цену, проходит 8 вопросов Purchase Check, получает AI-вердикт (купить / подождать / veto). Всё хранится локально. Никакой авторизации на старте.

---

## Стек

- **Frontend:** React (Vite) + TypeScript
- **Styling:** Tailwind CSS
- **Storage:** localStorage (без бэкенда)
- **AI:** Anthropic API (claude-sonnet-4-20250514) — вызов с фронта через proxy или напрямую с ключом в .env
- **Роутинг:** React Router v6
- **Деплой:** Vercel / Netlify (статика)

---

## Страницы и роуты

```
/                → Главная (Home)
/onboarding      → Онбординг (3 шага, только первый раз)
/check           → Purchase Check (8 вопросов)
/result          → Результат + AI-вердикт
/history         → История покупок
```

## Экраны — детальное описание

### 1. Онбординг `/onboarding`

Показывается один раз при первом визите. Результат сохраняется в localStorage под ключом `veto_profile`.

**Шаг 1 — Цель**
- Заголовок: "Зачем ты здесь?"
- Три варианта (radio-карточки):
  - 🛑 Трачу слишком много и хочу остановиться
  - 🧠 Хочу тратить осознаннее
  - 🎯 Коплю на конкретную цель

**Шаг 2 — Траты**
- Заголовок: "Сколько уходит на спонтанные покупки в месяц?"
- Слайдер: 0 — 100 000 ₽ (шаг 1000)
- Отображение значения над слайдером

**Шаг 3 — Цель накоплений (опционально)**
- Заголовок: "Есть цель для накоплений?"
- Два инпута: сумма + срок в месяцах
- Кнопка "Пропустить"

По завершению → редирект на `/`

---

### 2. Главная `/`

**Блок статистики** (берётся из localStorage):
- Сэкономлено: сумма всех "stopped" покупок
- Остановлено: количество "stopped" записей

**Форма добавления** (по центру экрана):
- Input: "Что хочу купить?" (text)
- Input: "Цена" (number) + валюта ₽
- Кнопка "Начать Purchase Check" → сохраняет данные в sessionStorage и переходит на `/check`

**Ссылка "История"** → `/history`

---

### 3. Purchase Check `/check`

Данные товара берутся из sessionStorage (`veto_current`).

**UI:**
- Прогресс-бар сверху (filled width = currentStep/8 * 100%)
- Текст "Вопрос N из 8"
- Карточка с вопросом:
  - Тег категории (ПОТРЕБНОСТЬ / ФИНАНСЫ)
  - Текст вопроса (крупный)
  - Две кнопки: ✅ Да / ❌ Нет
- Снизу фиксированная карточка: "Товар: [название] — [цена] ₽"

**Вопросы (в порядке):**

| # | Категория | Вопрос | "Правильный" ответ |
|---|-----------|--------|-------------------|
| 1 | ПОТРЕБНОСТЬ | Есть ли у меня уже что-то похожее? | Нет |
| 2 | ПОТРЕБНОСТЬ | Буду ли я пользоваться этим в ближайшие 48 часов? | Да |
| 3 | ПОТРЕБНОСТЬ | Это замена сломавшемуся? | Да |
| 4 | ПОТРЕБНОСТЬ | Думал ли я об этом больше недели? | Да |
| 5 | ПОТРЕБНОСТЬ | Купил бы я это по полной цене без скидки? | Да |
| 6 | ФИНАНСЫ | Укладывается ли это в мой бюджет? | Да |
| 7 | ФИНАНСЫ | Есть ли у меня финансовый запас в этом месяце? | Да |
| 8 | ФИНАНСЫ | Не пожалею ли я об этой сумме через неделю? | Да |

Ответы сохраняются в массив `answers: boolean[]`.

После 8-го вопроса → переход на `/result` (данные в sessionStorage).

---

### 4. Результат + AI `/result`

**Логика вердикта (локальная, до AI):**

Считаем score = количество "правильных" ответов из 8.

```
score >= 6  → verdict: "go"     → зелёный
score 3-5   → verdict: "wait"   → жёлтый  
score <= 2  → verdict: "veto"   → красный
```

**Верхняя часть экрана** — цветной блок (зелёный/жёлтый/красный):
- Иконка (✅ / ⏳ / 🛑)
- Заголовок:
  - go → "Выглядит обдуманно"
  - wait → "Подожди немного"
  - veto → "Veto!"
- Подзаголовок по вердикту

**Блок "Ты сохраняешь"** (если verdict != "go"):
- Крупно: сумма покупки
- Текст: "Если откажешься"

**Блок AI-анализа:**
- Заголовок "🤖 AI считает..."
- Состояния: loading (skeleton) → текст ответа
- AI получает промпт (см. ниже) и возвращает 2-3 предложения

**Кнопки внизу:**
- "На главную" → `/`
- "Посмотреть историю" → `/history`
- "Всё равно куплю" (серая, маленькая) → сохраняет как "bought", идёт на главную

**Сохранение в историю** происходит автоматически при открытии `/result`:

```typescript
interface HistoryEntry {
  id: string              // uuid
  name: string
  price: number
  answers: boolean[]
  score: number
  verdict: 'go' | 'wait' | 'veto'
  aiComment: string
  outcome: 'stopped' | 'bought' | 'pending'
  createdAt: string       // ISO date
}
```

---

### 5. История `/history`

**Статистика вверху:**
- Сэкономлено (сумма price всех stopped)
- Остановлено (count stopped)
- Куплено (count bought)

**Список записей** (новые сверху):
- Название + цена
- Вердикт-бейдж (цветной)
- Результат: "Остановлен" / "Куплен" / "Ожидает"
- Дата

## AI Промпт

```
Ты — Veto, умный помощник по осознанным покупкам. 
Отвечай кратко — 2-3 предложения. Без осуждения, без морализаторства. 
Как умный друг, который говорит честно.

Пользователь хочет купить: {name}
Цена: {price} ₽
Результат чека: {score} из 8 правильных ответов
Вердикт системы: {verdict}

Ответы на вопросы:
1. Есть похожее? {answers[0]}
2. Использую в 48ч? {answers[1]}
3. Замена сломанному? {answers[2]}
4. Думал > недели? {answers[3]}
5. Купил бы без скидки? {answers[4]}
6. В бюджете? {answers[5]}
7. Есть запас? {answers[6]}
8. Не пожалею через неделю? {answers[7]}

Профиль пользователя:
- Цель: {profile.goal}
- Средние спонтанные траты: {profile.monthlySpend} ₽/мес

Дай короткий персональный комментарий. 
Если verdict=go — одобри, но напомни про одну слабую точку если есть.
Если verdict=wait — объясни почему подождать и на сколько.
Если verdict=veto — скажи стоп, но мягко, с конкретной причиной.
```

## Что НЕ входит в MVP

- Авторизация и аккаунты
- Push-уведомления

- Rules / личные правила
- Статистика по эмоциям
- Мобильное приложение

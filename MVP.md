# VETO MVP: Техническое задание для Claude Code (Web SPA)

## 1. Контекст проекта (Project Context)
VETO — это социальный финансовый щит для зумеров, основанный на тренде "Loud Budgeting". Приложение превращает отказ от импульсивных трат в социальное достижение. 
Вместо того чтобы шерить покупки, пользователи шерят свои "отказы" (Vetos) и получают поддержку комьюнити (Respects).

**Главная задача MVP:** Проверить социальную механику в формате полноценного веб-приложения (Single Page Application). В MVP мы НЕ интегрируемся с реальными банковскими API. Все деньги и транзакции — это виртуальные цифры внутри базы данных.

## 2. Скоуп MVP (Strict MVP Scope)
В рамках этого спринта необходимо реализовать:
1.  **Landing Page:** Главная страница для объяснения концепции и привлечения пользователей.
2.  **Auth Flow:** Система регистрации и авторизации по Email / Паролю.
3.  **Web App Dashboard (Залогиненная зона):** Основной интерфейс приложения, куда пользователь попадает после входа.

**Что НЕ делаем в MVP (Out of Scope):**
- Реальные банковские переводы и API банков.
- Интеграция с Telegram или мобильными сторами (iOS/Android).
- Совместные (групповые) счета.
- Восстановление пароля (сброс по email) — для MVP достаточно базовой регистрации.

## 3. Рекомендуемый Технологический Стек
*Пожалуйста, используй именно этот стек при генерации кода:*

*   **Backend / API:** **Go (Golang)** + фреймворк Fiber (или Gin). 
    *   *Почему:* Высокая производительность, отлично работает для социальных лент, легко упаковывается в контейнер.
*   **Database:** PostgreSQL.
    *   *Почему:* Надежная реляционная база для хранения профилей, целей и ленты активности.
*   **Authentication:** JWT (JSON Web Tokens).
    *   *Почему:* Stateless-авторизация, идеально подходящая для связки Go + React.
*   **Frontend (Landing + App):** React.js + Vite + React Router + TypeScript.
    *   *Почему:* Быстрый роутинг между публичной страницей и защищенным дашбордом.
*   **Styling:** Tailwind CSS.
    *   *Почему:* Позволит быстро собрать UI в стиле "Dark Fintech" с поддержкой адаптивности.
*   **Infrastructure:** **Docker** + `docker-compose`.
    *   *Почему:* Единая среда для локальной разработки (Postgres + Go-API + Vite-Dev-Server).

## 4. UI/UX Дизайн Система (Design System)
Дизайн должен быть Mobile-First (так как ЦА сидит с телефонов), но корректно отображаться на десктопе. Стиль "Dark Fintech":
*   **Background:** Темно-синий/Угольный (`#10101A`). Все фоны темные.
*   **Text:** Белый (`#FFFFFF`) для заголовков, светло-серый (`#A0A0AB`) для вторичного текста.
*   **Primary Accent (Кнопка VETO / Успех):** Неоновый зеленый (`#00FF41`).
*   **Secondary Accent (Социальные функции / Alert):** Яркий коралловый (`#FF5E3F`).

## 5. Структура Базы Данных (Схема)
Создай миграции для следующих сущностей:
1.  `users`: id, email, password_hash, username, total_saved, created_at.
2.  `goals`: id, user_id, title (например, "Новый телефон"), target_amount, current_amount, status (active/completed).
3.  `vetos` (События отказа): id, user_id, goal_id, amount, description (например, "Кофе"), created_at.
4.  `respects` (Лайки): id, veto_id, from_user_id.

## 6. Требования к Фронтенду (Frontend Requirements)

### Public Routes (Публичная часть)
*   `GET /`: **Landing Page**. Hero-блок ("ЭКОНОМИТЬ — ЭТО НОВЫЙ ФЛЕКС"), описание функций. В шапке кнопки "Войти" и "Регистрация".
*   `GET /login` & `GET /register`: Страницы с формами авторизации. После успеха токен сохраняется в `localStorage` (или cookies), и происходит редирект на `/dashboard`.

### Protected Routes (Защищенная зона приложения)
На мобильных устройствах использовать нижнее навигационное меню (Bottom Navigation), на десктопе — боковой сайдбар.
*   `GET /dashboard` (Home/Щит):
    - Карточка текущей активной цели с прогресс-баром.
    - ОГРОМНАЯ кнопка "VETO" (The No-Button) по центру. 
    - При нажатии -> Модалка: "От чего отказываемся?" -> Инпуты: Сумма и Название (например, Кофе, 300) -> Кнопка подтвердить.
*   `GET /feed` (Анти-FOMO Лента):
    - Глобальная лента "Veto" от всех пользователей (для MVP).
    - Формат: "@alex ветировал 'Дорогая вечеринка' на 5000₽".
    - Кнопка "Respect" под каждой карточкой (увеличивает счетчик лайков).
*   `GET /profile`:
    - Данные пользователя (username).
    - Общая сэкономленная сумма (`total_saved`).
    - Форма для добавления/смены текущей финансовой цели.
    - Кнопка Logout.
